# フォーム仕様書

このドキュメントは Research-Form-Oasis のフォーム定義仕様を示します。

## 目的
- 研究用アンケートフォームのフィールド定義ルール
- フロントエンドとバックエンド（API/DB）が共通で扱うスキーマ
- CSV/Excel インポート時のマッピング規約

---

## フォーム定義（JSON スキーマ）

- 基本構造

```json
{
  "id": "form_yyyymmdd_001",
  "title": "アンケートタイトル",
  "description": "説明文（任意）",
  "fields": [
    {
      "id": "q1",
      "type": "single_choice",
      "label": "性別",
      "required": true,
      "options": ["男性","女性","回答しない"]
    }
  ],
  "meta": {
    "owner_id": "user_123",
    "created_at": "2026-04-28T12:00:00Z"
  }
}
```

### フィールドタイプ
- `single_choice` : 単一選択（ラジオ）
- `multi_choice`  : 複数選択（チェックボックス）
- `text_short`    : 短文（1行）
- `text_long`     : 長文（自由記述）
- `number`        : 数値（最小・最大バリデーション可能）
- `date`          : 日付
- `file`          : ファイルアップロード（管理者が許可）

### フィールド属性
- `id` : フォーム内で一意な識別子（英数字とアンダースコア）
- `label` : 表示ラベル
- `type` : 上記のフィールドタイプ
- `required` : true/false
- `options` : 選択肢配列（選択系のみ）
- `validation` : 追加バリデーション（正規表現や数値範囲）

## API 利用例（Go API 仕様）

- 回答登録

`POST /api/forms/{form_id}/responses`

Request JSON:

```json
{
  "answers": {
    "q1": "男性",
    "q2": "自由記述の回答"
  },
  "meta": { "anon": true }
}
```

Response: 201 Created

```json
{ "id": "resp_abcdef", "received_at": "2026-04-28T12:10:00Z" }
```

## CSV / Excel インポート規約
- ヘッダはフィールド `id` を用いる（例: `q1,q2,q3`）
- 日付は ISO8601 形式を推奨
- マルチ選択はセミコロン区切り（例: `opt1;opt3`）
- 空セルは null として扱う

サンプル CSV:

```
q1,q2,q3
男性,"自由記述テキスト",2026-04-01
女性,"別の回答",2026-04-02
```

## 管理者向け：辞書・メタデータ連携
- 自由記述解析のため、`dictionary_entry` テーブル（SurrealDB）にマスタ辞書を保存
- 辞書は `key`（正規形）と `variants`（表記揺れ配列）を持つ

例:

```json
{
  "key": "スマートフォン",
  "variants": ["スマホ","スマートフォン","スマートフォン\uFF08携帯\uFF09"]
}
```

## バリデーションとエラーハンドリング
- 400: フォーマット/必須チェック失敗
- 404: フォーム未登録
- 422: CSV インポート中の行単位エラー（エラー行を返却）

---

## フロントエンド実装メモ
- フォームビルダーは `fields` 配列を編集できる UI を提供
- 公開フォームは匿名送信を許可するため、認証は不要
- 管理画面は Google OAuth による保護を必須化
